Microshare HTTP API

Introduction

Welcome to Microshare’s API!

This API allows you to create and manage your data within Microshare’s data lake.

With Shares, you’ll be able to directly create, search, retrieve and delete data.

Rules will allow you to share this data or limit its visibility.

Robots are helpful scripts that run against your data to manipulate data structures, and perform actions initiated by the data itself.

API structures

Robots

Robots allow you to write scripts against your data stored as Shares in Microshare's data lake

recType string Record type using the dot notation format (eg. io.microshare.test)
id ObjectId An auto-generated unique identifier for the robot
name string Name of the robot
desc string Description of the robot
data JSON Object The data object contains a scope array with a list of scopes, a token under the auth key, the script of the robot under the script key, and a boolean isActive flag to activate or deactivate the robot

Rules

Rules allow you to limit the visibility of your data stored as Shares in Microshare's data lake

recType string Record type using the dot notation format (eg. io.microshare.test)
id ObjectId An auto-generated unique identifier for the rule
name string Name of the rule
desc string Description of the rule
data JSON Object The data object contains an operations array with a list of operations, and other variables defining the restrictions the rule is going to apply. More information is available below in the `POST /rule`

Shares

Shares represent your data stored in Microshare's data lake

recType string Record type using the dot notation format (eg. io.microshare.test)
id ObjectId An auto-generated unique identifier for the share
data JSON Object The data object is where your data is stored in our data lake

Authentication

Request and revoke tokens used to authenticate to our API

Request Token

Request an Authentification token required to use this API.

You will need to provide username, password, as well as a valid API Key. You can manage your API keys in our Keys Console.

The default scope is ALL:ALL, but you can replace it to request tokens that will allow limited operations. For example, if the scope is SHARE:READ the token will only allow you to read Shares.

  • Curl
  • HTTP
curl -X POST -H "Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW" "https://auth.microshare.io/oauth2/token?username={{username}}&password={{password}}&client_id={{apikey}}&grant_type=password&scope=ALL:ALL"
POST /oauth2/token?username={{username}}&password={{password}}&client_id={{apikey}}&grant_type=password&scope=ALL:ALL HTTP/1.1
Host: auth.microshare.io

Create one Rule

Create a new rule by recType.

You must include a record type using the dot notation format (eg. io.microshare.test).

Parameters in the data object are described below:

Parameter Type Description
operations array The operations to which the rule will apply. Possible values are Read, Write, Query, Delete, Execute, Policy
resourceType string The type of resource to which the rule will apply. Common value is objs for the data stored in shares, but could also be robos to limit access or share robots
resourceId ObjectId The id of the resource to which the rule will apply. Leave empty to apply the rule to all objects under the requested recType
ownerUser, ownerOrg, ownerApp, callingUser, callingOrg, callingApp, callingRole, callingLoc string These 8 parameters define the rule itself by publishing or restricting access to the requested objects. Possible values are *, &, ~, = or a specific string
